Step 02 — Reconcile
Bridge problem → approach → code
Once you recognize the pattern, reconcile means building the explicit mental bridge. Why this pattern? What's the invariant? Which template? Most people skip this and go straight to code — that's how bugs happen.
Step 03 — Recall
Build retrieval, not recognition
Recognition — seeing an answer and going "oh yeah" — is useless in an interview. You need active recall: producing the pattern cold, from a blank file, under pressure. Here's the system.

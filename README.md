# DSA Patterns — Pattern Breaker

A single, beginner-friendly study document for cracking coding interviews and contests. It teaches the most common DSA patterns with **intuition, templates, pitfalls, visualizations, and execution traces** so you can see the algorithm *run*, not just read about it.

## Quick start (no setup)
1. Open [`index.html`](./index.html) in your browser.
2. Stay on **Step 01 — Recognize** (the first pattern is preloaded).
3. Scroll down to the **Visualization 👁️** and **Execution Trace 🖥️** sections to see the algorithm’s step-by-step output.
4. Use **Glossary** for unknown terms and **Cheat Sheet** to compare patterns side-by-side.

## What you get in every pattern
- **Beginner Intuition (ELI5):** a simple mental model.
- **Recognition Signals:** keywords that reveal the pattern.
- **Template Skeleton:** the reusable code shape.
- **Common Pitfalls:** mistakes to avoid.
- **Worked Example:** a real LeetCode-style problem.
- **Visualization 👁️:** step-by-step state changes.
- **Execution Trace 🖥️:** printed log of how the code runs.

## Beginner study path (simple and focused)
**Week 1**
- Day 1–2: Sliding Window + Two Pointers  
- Day 3–4: Binary Search + BFS  
- Day 5–6: DFS / Backtracking + Prefix Sum  
- Day 7: Review with the Cheat Sheet + Pattern Drill

**Week 2**
- Pick 1 pattern per day. For each pattern:
  1. Read the intuition + signals  
  2. Write the template from memory  
  3. Walk through the visualization  
  4. Reproduce the execution trace on paper

## How to use this for interviews
- **First 2 minutes:** identify the pattern (use signals + cheat sheet).
- **Next 3 minutes:** restate the invariant and write the template skeleton.
- **Remaining time:** add problem-specific logic and test on sample input.

## No build or install
This is a static HTML document. There are no dependencies or build steps.
